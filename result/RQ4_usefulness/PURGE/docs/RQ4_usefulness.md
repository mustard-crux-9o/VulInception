# RQ4: Usefulness Analysis — Reproduction Guide

This document mirrors **RQ4** in the paper: demonstrating that VulInception-identified
vulnerability members serve as a precise forget set for PURGE-based unlearning.

## Experimental setup

### Unlearning method

**PURGE** — reinforcement unlearning via Group Relative Policy Optimization (GRPO).
Implementation: `src/purge_code.py` with `reward=code_binary` (default).

### Forget-set selectors

| Selector | Preprocessor | Hydra config |
|----------|--------------|--------------|
| VulInception | `src/prepare_code_data.py` | `dataset=pile` or `stack`, `paths=code_default` |
| Random | `scripts/prepare_random_forget.py` | `dataset=pile_random` / `stack_random`, `paths=code_forget_sets` |
| Gotcha | `scripts/prepare_gotcha_forget.py` | `dataset=pile_gotcha` / `stack_gotcha` |
| Groundtruth | `scripts/prepare_groundtruth_forget.py` | `dataset=pile_groundtruth` / `stack_groundtruth` |

Prepare all sets at once:

```bash
bash scripts/prepare_all_forget_sets.sh
```

### Target models

| Model | Pretraining corpus | Hydra `model=` |
|-------|-------------------|----------------|
| GPT-J-6B | The Pile | `gpt_j_6b` |
| StarCoderBase-7B | The Stack | `starcoderbase_7b` |

Training hyperparameters: `training=code_full` (20 epochs, GRPO with 4 generations per prompt).
For faster rollouts on A100, use `training=code_a100_single` (1 GPU) or `training=code_full_fast` (2 GPU).
Both set `VLLM_USE_V1=1` automatically when vLLM colocate mode is enabled.

### Evaluation benchmarks

1. **VMI-Bench / SecurityEval** — vulnerability-relevant prefixes from VMI-Bench;
   completions scanned by Bandit → **Vulnerability Rate (VR)**.
2. **Unseen 2026 CVEs** — VulInception benchmark (`../Unseen2026CVE/dataset.jsonl`); Bandit VR.
3. **HumanEval** — Pass@1 with greedy decoding to measure retained coding ability.

Benchmarks are under `benchmarks/`; Unseen 2026 CVE is from VulInception (not in this repo).

## End-to-end workflow

Benchmarks are bundled under `benchmarks/` (see `benchmarks/README.md`).

```bash
# 0. Environment
conda activate purge && pip install -r requirements.txt
pip install -e benchmarks/humaneval
cp scripts/env.example.sh scripts/env.sh   # optional; defaults point to benchmarks/

# 1. Data
bash scripts/prepare_all_forget_sets.sh

# 2. Train (example: all selectors for one model)
bash scripts/run_rq4_train_all.sh gpt_j_6b

# Or train individually:
bash scripts/run_train_forget_set.sh pile gpt_j_6b
bash scripts/run_train_forget_set.sh stack starcoderbase_7b

# Accelerated training (vLLM colocate, requires VLLM_USE_V1=1):
# TRAINING=code_a100_single bash scripts/run_train_forget_set.sh stack_random starcoderbase_7b paths=code_forget_sets
source scripts/env.sh
bash scripts/run_rq4_usefulness_eval.sh gpt_j_6b
```

  `models/code/` or set `*_MODEL` variables in `scripts/env.sh`.
